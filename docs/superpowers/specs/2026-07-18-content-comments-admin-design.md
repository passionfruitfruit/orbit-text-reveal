# 个人网页内容中心、留言与统一后台设计

## 目标

在现有 Orbit 文字动画和平台入口之后，一次完成网站的主要功能：

1. 将 Bilibili 投稿、GitHub 项目和博客文章放进一条可排序的混合内容流。
2. 提供带双重公开许可、回复、联系方式隐私和站长审核的留言系统。
3. 新增独立管理页，统一配置 Orbit、平台入口、内容来源、内容排序、博客和留言。
4. 使用 Sites 托管的持久化数据保存配置、同步结果、博客和留言。

现有 Orbit 组件的内部时间轴、文字排版、精确回中、滚动缩放语义和开发者页面保持兼容。

## 页面结构

正式页面从上到下为：

1. Orbit 文字动画首屏。
2. 现有平台入口卡片。
3. 混合内容流。
4. 留言与公开讨论区。

正式页面不出现后台控件。管理入口位于独立的 `admin.html`。现有 `dev.html` 继续作为 Orbit 组件开发预览页，不承担网站内容管理职责。

## 视觉与响应式布局

延续当前偏粉的米白背景、纯黑正文和 `18px` 圆角语言，不增加新的主色主题。

### 混合内容卡片

- Bilibili、GitHub 和博客混合在同一条单列内容流中，不自动分组，也不提供分类标签页。
- 桌面端和移动端始终只有一列，桌面端左右保留装饰空间。
- 每页最多 10 条，超过后显示上一页、页码和下一页。
- 管理后台可对任意类型的内容统一拖动排序。
- 整张卡片可点击，跳转到 Bilibili 投稿、GitHub 项目、外部博客或站内文章详情。
- Bilibili 卡片左侧显示封面，右侧显示标题、简介、发布时间和可用的播放信息。
- GitHub 卡片左侧显示项目来源标识，右侧显示项目名、简介、主要语言、更新时间和 Star 数。
- 博客卡片支持可选封面，显示标题、简介和发布时间。
- 极窄屏幕缩小左侧媒体区域并允许正文自然换行，不产生横向滚动。

### 留言样式

- 使用标准评论布局：圆形头像占位、昵称、时间、正文和回复操作。
- 回复在根留言下缩进展示，但不使用卡片套卡片。
- 留言表单包含昵称、留言正文、可选联系方式和“允许公开展示”复选框。
- 回复表单包含昵称、回复正文和可选联系方式。
- 表单具有明确的提交中、成功、失败和校验状态。

## 内容来源与同步

### 默认来源

- Bilibili UID：`496633495`。
- GitHub 用户名：`passionfruitfruit`。
- 博客初始为空。

后台允许添加、停用和删除多个 Bilibili UID 与 GitHub 用户名。访客浏览器不直接请求第三方平台；服务端负责同步并把标准化结果保存到数据库。

### 同步规则

- 管理员可在后台手动触发单个账号或全部账号同步。
- Bilibili 同步拉取公开视频投稿的标题、简介、封面、链接、发布时间和接口可提供的统计信息。
- GitHub 同步拉取公开仓库的名称、简介、链接、主要语言、更新时间和 Star 数。
- 同步使用平台类型与外部内容 ID 幂等更新，不重复创建内容。
- 已由管理员隐藏或手工排序的内容在再次同步后保持隐藏状态和排序位置。
- 已从来源消失的内容不自动删除，后台标记为“来源中未找到”，由管理员决定隐藏或删除。
- 同步失败不清空已有内容；后台显示最近成功时间和可理解的失败信息。
- 未配置平台令牌时使用公开接口；GitHub 令牌等可选凭据只能保存在托管环境的私密变量中。

### 混合排序与显示

每条内容具有 `visible` 与 `sort_order`。公开接口只返回 `visible = true` 的内容，并严格按 `sort_order`、发布时间和稳定 ID 排序。后台可：

- 单独显示或隐藏投稿、项目和博客。
- 跨类型拖动排序。
- 批量显示或隐藏。
- 删除手工博客；同步内容默认只隐藏，明确确认后才能从本地记录删除。

## 博客

博客支持两种条目：

1. 站内文章：标题、简介、正文、可选封面、发布时间和公开状态，点击进入站内详情页。
2. 外部文章：标题、简介、外部 URL、可选封面、发布时间和公开状态，点击跳转外部页面。

博客初始为空，前台在没有博客时不显示空占位。站内正文第一版使用安全的纯文本分段格式，不执行管理员输入的 HTML 或脚本。

## 留言公开与隐私规则

### 根留言

根留言是否出现在公开页面必须同时满足：

```text
留言者允许公开 = true
并且
管理员批准公开 = true
并且
留言未被删除
```

任一条件不满足，根留言及其全部回复都不会出现在公开接口。

### 回复

- 回复的公开范围继承根留言，回复者不能扩大根留言的公开范围。
- 每条回复仍需管理员单独批准后才可展示。
- 根留言被隐藏、取消公开或删除时，其全部回复立即从公开页面消失。
- 管理员可以“站长”身份回复，站长回复仍遵守根留言的公开范围。

### 联系方式

- 联系方式为可选字段，可以填写邮箱、QQ、社交账号或其他文本。
- 联系方式只通过受保护的后台接口返回。
- 公开留言与回复接口永远不包含联系方式字段，即使管理员误操作也不能公开。
- 软删除期间私人联系方式仍仅在后台保留，以便恢复；永久删除时一并彻底删除对应私人联系方式。

### 删除与审核

- 后台提供待审核、已公开、仅后台可见和已删除视图。
- 管理员可批准公开、取消公开、软删除和恢复根留言或回复。
- 管理员可执行永久删除；永久删除根留言时级联删除全部回复和联系方式。
- 对同一来源的短时间重复提交进行服务端频率限制，并使用隐藏的机器人诱捕字段降低垃圾提交。
- 留言正文、昵称和联系方式都执行长度限制与纯文本转义。

## 管理后台

后台是独立页面，使用单一管理员口令登录。口令只存放在 Sites 私密环境变量中，服务端验证成功后签发短时、`HttpOnly`、`Secure`、`SameSite=Strict` 的会话 Cookie。前端代码和数据库中都不保存明文口令。

后台包含以下栏目：

1. **首屏文字**：管理 Orbit 文本队列、时间、布局和外观，复用现有标准化与预览能力。
2. **平台入口**：管理第一部分卡片的名称、简介、图标、图标方向、链接或复制动作和排序。
3. **内容来源**：管理 Bilibili UID、GitHub 用户名、启用状态、同步和最近同步结果。
4. **内容展示**：统一管理投稿、项目和博客的显示状态与混合顺序。
5. **博客**：新增、编辑、预览、发布、隐藏和删除站内或外部文章。
6. **留言**：审核、公开、隐藏、回复、查看私人联系方式、软删除、恢复和永久删除。

所有管理写操作都必须在服务端再次校验管理员会话，不能只依赖前端隐藏按钮。

## 数据模型

使用 Sites D1 保存结构化数据，不使用浏览器本地存储作为事实来源。主要记录为：

- `site_settings`：Orbit 与平台入口的版本化 JSON 配置。
- `source_accounts`：平台、账号标识、启用状态和同步状态。
- `content_items`：标准化投稿、项目和博客摘要、显示状态与混合排序。
- `blog_posts`：站内正文或外部 URL。
- `comments`：根留言与回复关系、正文、公开许可、审核状态、删除状态和私人联系方式。
- `admin_sessions`：只保存会话摘要和过期时间，不保存管理员口令。
- `submission_limits`：经过不可逆摘要处理的限流标识与时间窗口。

数据库迁移文件随项目保存。公开查询和后台查询使用不同的数据投影，公开投影在服务端明确排除私人字段。

## 接口边界

公开接口：

- 获取当前公开网站配置。
- 获取混合内容分页，每页最大 10 条。
- 获取单篇公开站内博客。
- 获取公开根留言及已批准回复。
- 提交根留言。
- 向公开根留言提交回复。

后台接口：

- 登录、检查会话和退出。
- 读取与保存所有网站配置。
- 管理来源账号并触发同步。
- 管理内容显示、顺序和博客。
- 查看含私人联系方式的留言数据。
- 审核、回复、隐藏、恢复和删除留言。

所有接口返回稳定的错误代码和适合界面展示的中文信息。第三方平台错误与内部异常不向访客暴露凭据或堆栈。

## 初始数据与兼容迁移

- 首次启动时，将当前 `config.js` 的 Orbit 配置和现有三张平台入口导入 `site_settings`。
- 在数据库暂无配置或接口不可用时，正式页继续使用当前静态配置作为只读回退，避免首屏空白。
- 默认创建 Bilibili `496633495` 和 GitHub `passionfruitfruit` 两个启用来源。
- 博客与留言初始为空。
- 保留 `config.js`、现有组件接口和 `dev.html`，避免破坏组件的独立使用方式。

## 错误处理与可用性

- 内容和留言加载失败时，各区块独立显示重试状态，不影响 Orbit 与平台入口。
- 第三方同步失败保留上一次成功数据。
- 管理保存使用明确的保存中、成功和失败反馈，重复提交按钮在请求期间禁用。
- 管理员会话失效时返回登录页，不丢失浏览器中尚未提交的编辑草稿。
- 前台和后台均支持键盘操作、可见焦点、语义标签和减少动态效果偏好。

## 验证标准

- Node 合同测试覆盖配置标准化、内容混排、分页、同步幂等、公开数据投影、留言双重许可、回复继承、级联隐藏、永久删除和后台授权。
- 浏览器测试覆盖桌面与移动布局、单列内容、每页 10 条、翻页、内容卡跳转、留言提交、回复流程和后台主要编辑流程。
- 安全回归测试证明公开接口不包含联系方式、未授权管理写操作被拒绝、第三方失败不删除旧数据。
- 运行完整 `npm test`、部署构建、浏览器断言页、控制台检查和 `git diff --check`。
- 使用真实本地服务验证数据刷新后仍然存在，而不是只验证静态组件。

## 发布

完成并验证后通过 Sites 发布。正式部署使用 D1 持久化绑定和私密环境变量；管理员口令不进入 Git、前端资源、测试快照或日志。发布前需要由站长设置正式管理员口令。
