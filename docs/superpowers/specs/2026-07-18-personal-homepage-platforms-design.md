# 个人网页首屏过渡与平台入口设计

日期：2026-07-18

## 范围

本阶段只完成两部分：

1. 用户首次向下浏览时，首屏 Orbit 文本组件跟随滚动向页面上部移动并缩小；过渡完成后恢复自然页面滚动。返回页面顶部时完整反向恢复。
2. 增加“其他平台”入口区，以及对应的独立开发者配置能力。

GitHub 项目、B 站投稿、博客、留言页和照片展示不在本阶段实现，但页面结构应允许这些区块以后继续向下添加。

本规格明确取代 `2026-07-18-personal-homepage-hero-design.md` 中“不得新增第二屏”的阶段性限制；此前确定的 Orbit 文本、颜色、连续响应式尺寸、窗口缩放桥接和动画语义继续有效。

现有 Orbit 文字展开、收回、逐行路径、尺寸安全重排和精确回中语义保持不变。正式页不出现开发控件，配置工具继续只存在于 `dev.html`。

## 首屏滚动交互

采用“黏性首屏舞台 + 连续滚动进度”的实现：页面拥有真实滚动距离，但在开场过渡期间首屏视觉舞台保持固定，因此用户不会看到整页提前向上离开。

正式结构使用一个 `.intro-sequence` 容器和一个高度为 `100svh`、`position: sticky; top: 0` 的 `.intro-scene`。`.intro-sequence` 的总高度为 `calc(100svh + var(--intro-travel))`，其中：

```css
--intro-travel: clamp(520px, 85svh, 900px);
```

额外的 `--intro-travel` 就是完整过渡距离。进度使用当前容器滚动距离除以该值并钳制到 `0–1`。当滚动超过这段距离时，黏性场景自然解除固定，Orbit 和平台区作为同一页面场景一起向上滚走，不进行 `fixed` / `absolute` 瞬时切换。

- 用户滚轮向下、触控板向上推或手指向上滑时，滚动距离连续映射到 `0–1` 的开场进度。
- 进度 `0` 时，Orbit 组件保持当前视口中央位置和 `100%` 外部比例。
- 进度 `1` 时，Orbit 组件仍严格水平居中，移动到页面上部，并缩小到初始外部比例的 `65%`。
- 移动和缩放使用同一个连续进度，不设置桌面/移动断点，不重新启动 Orbit 内部文字时间轴。
- 用户停止输入 `140ms` 后，若开场进度严格位于 `0–1` 之间，则平滑补完到最近端点；恰好为 `0.5` 时按最近滚动方向决定。新的用户输入立即取消补完，避免与用户争抢滚动控制。
- 开场进度完成后，黏性首屏到达容器底部并自然解除，后续滚动直接带动整个场景向上移动；不存在额外的滚动锁或突然释放。
- 用户滚回页面顶部时，所有开场状态按完全相反的路径恢复；再次向下浏览可以重新播放。
- 在过渡未完成时，不允许横向偏移、页面横向滚动或 Orbit 动画索引意外重置。
- 极高惯性滚动可以越过过渡终点。进度在 `1` 处钳制，多余滚动距离继续交给文档流，不截断或吞掉用户输入。

页面必须移除当前 `html, body { overflow: hidden }` 的首屏限制，改为允许纵向滚动并继续禁止横向溢出。使用浏览器原生 `scroll` 位置驱动进度，不手动阻止 `wheel` 或 `touchmove`，从而保留触控板惯性和移动端触摸滚动。

Orbit 的最终顶部位置使用连续视口公式：

```css
--intro-orbit-center-y: clamp(18svh, calc(15svh + 3vw), 20svh);
```

窄屏终点锁定在安全视口高度的 `18%`，宽屏终点锁定在 `20%`，中间连续变化，并保证最长文本在缩小状态下仍不被页面边缘裁切。

实现继续使用现有 `--orbit-page-y` 和 `--orbit-page-scale` 外部变量：`--orbit-page-y` 在像素空间把当前 `50%` 中心连续映射到 `--intro-orbit-center-y`，`--orbit-page-scale` 从 `1` 连续映射到 `0.65`。`top: 50%` 和 `translate(-50%, -50%)` 的基础定位保持不变，避免终点切类造成跳变。

## 平台区出现动画

平台入口区在 Orbit 接近目标位置时开始出现：

- 开场进度约 `62%` 前，平台内容保持透明并位于目标位置下方约 `24px`。
- `62%–88%` 之间逐渐变为不透明并向上移动。
- 接近结束时最多越过目标位置约 `5px`，随后轻轻落回 `0px`。
- 平台区位于 `.intro-scene` 内；黏性阶段跟随同一场景显现，过渡完成后随解除黏性的场景进入正常文档滚动。
- 反向滚回顶部时，透明度、位移和轻微回弹按相反顺序撤销。
- 多张卡片使用很短的顺序错开，最后一张相对第一张最多延迟 `0.06` 开场进度，避免显得拖沓。

系统启用 `prefers-reduced-motion: reduce` 时，取消回弹和自动平滑补完；Orbit 与平台区直接切换到与当前滚动位置对应的稳定状态。

## 平台区布局

平台区标题和卡片网格放在不带外框的页面区块中：

- 宽屏使用两列卡片。
- 窄屏使用一列卡片。
- 平台容器使用连续宽度公式 `min(calc(100vw - 32px), calc(52vw + 121.6px), 1120px)`。它在 `320px` 视口下保留左右各 `16px`，在 `1920px` 视口下宽 `1120px`、左右各保留 `400px`，中间连续变化，为后续装饰长期保留空间。
- `900px` 及以上使用两列；低于 `900px` 使用一列。列数是离散布局选择，容器宽度本身不发生断点跳变。
- 卡片为横向圆角长方形，圆角 `8px`，不嵌套在其他卡片中。
- 卡片最小高度为 `132px`，内边距为 `22px 24px`；移动端为 `120px` 和 `18px`。网格行列间距为 `18px`。
- 卡片包含一个平台图标和一个左对齐文本区。大字为平台名称，小字为简介。
- 桌面图标使用稳定的 `56px × 56px` 区域，移动端为 `48px × 48px`。标题使用 `24px/1.2`，描述使用 `14px/1.55`；移动端标题为 `21px`。
- 图标在左时，文本区位于图标右侧并保持左对齐；图标在右时，文本区从卡片左侧开始，图标贴近右侧。
- 卡片高度和图标区域使用稳定尺寸，悬停、复制反馈或长文本不得造成网格跳动。
- 桌面端提供轻微悬停抬升和边框变化；触屏端不依赖悬停才能理解或操作。
- 平台区从场景顶部 `clamp(250px, 36svh, 380px)` 处开始，底部至少保留 `48px`。在 `320×700` 下三张单列卡片可以进入场景范围，继续滚动时不会被黏性容器裁切。
- 卡片使用接近白色的半透明表面、低对比黑色边框和纯黑文字；平台品牌颜色主要由图标承担，页面继续沿用现有淡粉米白背景。

## 卡片行为

每张卡片只有一种动作：

- `link`：点击后打开配置的个人主页链接。外部链接默认在新标签页打开，并使用安全的 `noopener noreferrer`。
- `copy`：点击后复制配置文本到剪贴板。成功后在卡片内部显示简短“已复制”状态，随后自动恢复；复制失败时显示可理解的失败提示，不静默失败。

所有卡片都可以通过键盘聚焦并用 Enter 或 Space 触发。链接卡片保留真实链接语义，复制卡片使用真实按钮语义。

## 正式配置

`config.js` 保留现有 `animationConfig`，并新增独立的 `platformConfig` 导出。平台数据不混入 Orbit 的 `normalizeConfig()`，避免破坏现有组件配置合同。

每条平台配置使用以下结构：

```js
{
  id: 'bilibili',
  title: 'bilibili',
  description: '视频、动态与我的公开投稿',
  icon: './assets/platforms/bilibili.svg',
  iconSide: 'left',
  action: {
    type: 'link',
    value: 'https://space.bilibili.com/496633495?',
    newTab: true
  }
}
```

字段约束：

- `id`：稳定且唯一，用于 DOM 和配置编辑。
- `title`、`description`：非空字符串。
- `icon`：本地相对路径或有效图片 URL；正式首批图标优先保存为本地品牌资源，避免页面运行依赖第三方 CDN。
- `iconSide`：仅接受 `left` 或 `right`。
- `action.type`：仅接受 `link` 或 `copy`。
- `action.value`：链接地址或待复制文本。
- `action.newTab`：只对链接动作有效。

无效条目在开发者页显示校验错误，不进入正式渲染。平台数量不设固定上限。

## 开发者配置页

`dev.html` 新增独立“平台入口”编辑区，但不得把任何编辑控件放入正式页。

编辑能力包括：

- 新增、删除、复制和排序平台卡片。
- 编辑平台名称、简介和图标路径。
- 选择图标在左或在右。
- 选择链接或复制动作。
- 编辑链接地址或待复制文本，并设置链接是否新标签打开。
- 实时预览宽屏两列与窄屏一列结果。
- 单独复制或下载平台配置；现有 Orbit 动画配置的复制和下载格式保持向后兼容。

平台配置的标准化和编辑操作放入独立模块，避免继续扩大 `src/dev-app.js` 的职责。

## 文件边界

- `index.html`：增加开场舞台结构和平台区语义标记。
- `main.js`：启动 Orbit、平台渲染和开场滚动控制器。
- `config.js`：保留 `animationConfig`，新增 `platformConfig`。
- `src/base.css`：正式页布局、黏性首屏、平台网格和动画状态。
- 新平台配置模块：标准化平台数据并提供纯函数操作。
- 新平台渲染模块：只负责生成链接或复制卡片及反馈。
- 新开场控制模块：只负责把滚动位置映射为稳定进度和自动补完。
- `dev.html` / `src/dev.css`：平台编辑器界面。
- `src/dev-app.js`：协调现有 Orbit 编辑器与新的平台编辑模块，不承担平台数据细节。

## 验收

- 开场进度在滚轮、触控板和触摸滚动下连续变化，不出现断点。
- 半程停止后自动补完到最近端点；补完过程中重新滚动会立即把控制权交还用户。
- Orbit 从中央移动到顶部区域时始终水平居中，终点比例精确为 `0.65`。
- Orbit 内部时间轴、当前文本索引和尺寸安全重排不因页面过渡重新开始。
- 平台区在 Orbit 接近终点时渐显，出现一次轻微上越和回落。
- 过渡完成后浏览器恢复普通滚动；滚回顶部时过渡完整反向恢复。
- 反向滚动期间平台区按相反进度淡出并下移，回到进度 `0` 时不可见且不可聚焦。
- 桌面两列容器左右保留明显装饰空间；窄屏一列且无横向滚动。
- Bilibili 和 QQ 链接地址精确匹配需求；邮箱卡片复制精确文本 `mail@zhang.jx.cn`。
- 图标左右、卡片数量、文字、动作类型和动作值都能通过正式配置与开发者页修改。
- 链接和复制卡片具备正确键盘语义，复制成功与失败均有反馈。
- 浏览器测试确认 Tab 聚焦顺序、Enter/Space 激活、复制成功提示和复制失败提示。
- 减少动态效果模式下无回弹和强制平滑动画。
- Node 合同测试先失败后通过；浏览器测试覆盖正向、反向、断点布局、复制行为和控制台错误。
- 原有 Node 测试、Orbit 浏览器断言和开发者配置回归继续通过。
- `git diff --check` 无错误。
