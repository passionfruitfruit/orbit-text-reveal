# 个人网页首屏 Orbit 响应式设计

日期：2026-07-18

## 范围

正式页面首屏只展示现有 `<orbit-text-reveal>` 组件。不得新增导航、按钮、滚动提示、第二屏或装饰图形。开发者配置页继续独立存在，现有动画路径、全局时间轴、暂停恢复、减少动态效果与精确回中语义不变。

## 文本

按以下顺序循环：

1. `hello:)`
2. `欢迎！`
3. `粉骨碎身浑不怕\n要留清白在人间`
4. `真的英雄：不是打败世界的傲慢，而是勇敢的守住内心的天真`
5. `你来这里干嘛\\（≧▽≦）/`
6. `（别光看屏幕啦！`

只有第三条强制手动换行。其余文本允许根据可用宽度自动换行。

## 位置与颜色

- 组件舞台在真实视口内严格水平、垂直居中。
- 页面与组件舞台背景统一为极淡粉色米白 `#f7f2ef`。
- 文字和圆球均为纯黑 `#000000`。
- 页面使用动态视口高度，并保留安全的 `svh` 回退，避免移动端浏览器栏导致视觉中心偏移。

## 连续响应式宽度

响应式计算同时使用视口宽度与高度，让窄长的桌面窗口也会逐渐接近移动端构图：

- `1920x1080`：左留白、文本舞台、右留白为 `3/4/3`，舞台宽度为 `40vw`。
- `320x700`：比例为 `1/7/1`，舞台宽度为 `77.7777778vw`。
- 两个参考比例之间按 `vw + vh` 线性连续插值；更宽或更窄时锁定端点。
- 不使用媒体查询或 JavaScript 断点，窗口宽度、高度和宽高比的任意连续变化都必须得到连续结果。

CSS 等价公式：

```css
width: clamp(
  40vw,
  calc(26.9230769vw + 23.2478632vh),
  77.7777778vw
);
```

左右留白始终各占剩余宽度的一半，因此舞台保持精确水平居中。

## 连续响应式字号

- `320x700` 视口字号为 `19px`，为十个汉字、圆球、间距和两侧 16px 安全区共同留出空间。
- `1920x1080` 视口字号为 `64px`。
- 字号与舞台宽度使用相同的宽高比原则连续插值，超出范围锁定端点。

```css
font-size: clamp(19px, calc(2.431891vw + 1.6025641vh), 64px);
```

内部 `maxWidth` 不再成为宽屏上限，由真实舞台宽度、16px 安全边距、圆球和间距共同约束。组件读取流式 CSS 字号时，若自定义属性使用 `calc()` 或 `clamp()`，必须采用浏览器已计算的像素字号进行文字测量。最窄正式视口必须在自动换行时保留至少 10 个中文字符的单行容量；组件如需自动拟合，只能缩小到满足边界，不得造成水平偏移或裁切。

## 拖动窗口时的连续视觉缩放

组件继续遵守“正式几何只在当前文字收回并回到中心后安全重排”的动画语义，但不能让旧几何在等待期间保持固定字号。

- `ResizeObserver` 每次观察到有效尺寸变化时，立即读取当前目标字号和新舞台中心。
- 在正式几何尚未安全重建时，对整个 `.visual` 层应用临时平移与等比缩放：旧几何中心映射到新舞台中心，文字、圆球、间距和当前运动位置作为一个整体连续变化。
- 临时缩放以当前几何中心为变换原点，不重新启动时间轴、不改变当前进度、不增加索引事件。
- 当前文字完成反向收回并回到中心后，组件用最终尺寸重建几何，并在同一中心状态清除临时变换；切换前后的圆球中心和可见尺寸误差不超过 1px。
- 连续拖动产生的多次尺寸通知只保留最新目标；不得在每个通知上重启动画或累积变换。
- 开发者页面、普通固定尺寸嵌入、暂停恢复、减少动态效果和精确回中语义保持兼容。

## 验收

- 在 `320x700`、`390x844`、`960x1080`、`1280x800` 和 `1920x1080` 下验证宽高比公式与连续性。
- 固定宽度、只改变高度时，舞台宽度和字号也必须按公式连续变化。
- 连续模拟至少 20 个相邻视口尺寸，任意相邻两帧的目标宽度和目标字号不得出现断点跳变。
- 在桌面和移动真实视口下，组件边界中心与视口中心的水平、垂直误差均不超过 1px。
- `320px` 下验证至少 10 个等宽中文字符可形成一行，页面无横向滚动。
- 动画展开、停留和收回阶段分别触发连续尺寸变化时，临时视觉中心误差不超过 1px，字号变化方向与目标公式一致，且不产生额外重启或索引切换。
- 安全重排前后验证圆球中心、视觉比例和动画状态连续，不出现可见的突然放大或缩小。
- 六条文本内容、顺序和唯一手动换行点必须精确匹配。
- 背景、文字、圆球颜色必须精确匹配规格。
- Node 合同测试、浏览器断言和控制台检查全部通过。
- 原有展开、收回、逐行蛇形路径与圆球精确回中回归测试继续通过。
