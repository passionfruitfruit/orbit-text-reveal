# 个人网页首屏 Orbit 响应式设计

日期：2026-07-18

## 范围

正式页面首屏只展示现有 `<orbit-text-reveal>` 组件。不得新增导航、按钮、滚动提示、第二屏或装饰图形。开发者配置页继续独立存在，现有动画路径、全局时间轴、暂停恢复、减少动态效果与精确回中语义不变。

## 文本

按以下顺序循环：

1. `hello:)`
2. `欢迎！`
3. `粉骨碎身浑不怕\n要留清白在人间`
4. `真的英雄：不是打败世界的傲慢，而是勇敢的守住内心的天真`
5. `你来这里干嘛\\（≧▽≦）/`
6. `（别光看屏幕啦！`

只有第三条强制手动换行。其余文本允许根据可用宽度自动换行。

## 位置与颜色

- 组件舞台在真实视口内严格水平、垂直居中。
- 页面与组件舞台背景统一为极淡粉色米白 `#f7f2ef`。
- 文字和圆球均为纯黑 `#000000`。
- 页面使用动态视口高度，并保留安全的 `svh` 回退，避免移动端浏览器栏导致视觉中心偏移。

## 连续响应式宽度

以视口宽度 `V` 为基准：

- `V <= 320px`：左留白、文本舞台、右留白为 `1/7/1`，舞台宽度为 `77.7777778vw`。
- `V >= 1920px`：比例为 `3/4/3`，舞台宽度为 `40vw`。
- `320px < V < 1920px`：舞台宽度线性连续插值，无媒体查询跳变。

CSS 等价公式：

```css
width: max(
  40vw,
  min(77.7777778vw, calc(32.4444444vw + 145.0666667px))
);
```

左右留白始终各占剩余宽度的一半，因此舞台保持精确水平居中。

## 连续响应式字号

- `320px` 视口字号为 `19px`，为十个汉字、圆球、间距和两侧 16px 安全区共同留出空间。
- `1920px` 视口字号为 `64px`。
- 中间线性插值，超出范围锁定端点。

```css
font-size: clamp(19px, calc(2.8125vw + 10px), 64px);
```

内部 `maxWidth` 不再成为宽屏上限，由真实舞台宽度、16px 安全边距、圆球和间距共同约束。组件读取流式 CSS 字号时，若自定义属性使用 `calc()` 或 `clamp()`，必须采用浏览器已计算的像素字号进行文字测量。最窄正式视口必须在自动换行时保留至少 10 个中文字符的单行容量；组件如需自动拟合，只能缩小到满足边界，不得造成水平偏移或裁切。

## 验收

- 在 `320px`、若干中间宽度和 `1920px` 下验证宽度公式与连续性。
- 在桌面和移动真实视口下，组件边界中心与视口中心的水平、垂直误差均不超过 1px。
- `320px` 下验证至少 10 个等宽中文字符可形成一行，页面无横向滚动。
- 六条文本内容、顺序和唯一手动换行点必须精确匹配。
- 背景、文字、圆球颜色必须精确匹配规格。
- Node 合同测试、浏览器断言和控制台检查全部通过。
- 原有展开、收回、逐行蛇形路径与圆球精确回中回归测试继续通过。
